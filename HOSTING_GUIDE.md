# Руководство по хостингу портфолио

## Проблема с 12 ГБ медиафайлов

**GitHub Pages НЕ подходит для 12 ГБ медиафайлов** по следующим причинам:
- Лимит репозитория GitHub: 100 ГБ (но это не для хостинга)
- GitHub Pages не предназначен для больших медиафайлов
- Загрузка будет очень медленной
- Может быть заблокирован за превышение лимитов

## Решения для хостинга больших медиафайлов

### Вариант 1: GitHub Pages + Внешнее хранилище (РЕКОМЕНДУЕТСЯ)

**Структура:**
- Код сайта (HTML, CSS, JS) → GitHub Pages (бесплатно)
- Медиафайлы → Внешнее хранилище

**Варианты внешнего хранилища:**

#### A. Cloudinary (бесплатный тариф)
- 25 ГБ хранилища бесплатно
- Автоматическая оптимизация изображений
- CDN для быстрой загрузки
- Поддержка видео

#### B. AWS S3 + CloudFront
- Очень дешево (около $0.023 за ГБ/месяц)
- 12 ГБ ≈ $0.28/месяц
- Быстрая загрузка через CDN

#### C. Google Cloud Storage
- Аналогично AWS
- Хорошая интеграция

#### D. Netlify / Vercel
- Бесплатный хостинг для статических сайтов
- Но медиафайлы все равно нужно хранить отдельно

### Вариант 2: Полный хостинг на платформе для медиа

#### A. Netlify (рекомендую для портфолио)
- Бесплатный тариф: 100 ГБ трафика/месяц
- Можно загрузить медиафайлы
- Автоматический деплой из Git
- Быстрый CDN

#### B. Vercel
- Аналогично Netlify
- Хорошо для статических сайтов

#### C. GitHub Pages + Git LFS (не рекомендуется)
- Git LFS позволяет хранить большие файлы
- Но все равно будет медленно
- Лимиты GitHub могут быть превышены

## Рекомендуемый подход

### Для начала (быстрый старт):
1. **Код на GitHub Pages** - бесплатно, быстро
2. **Медиафайлы на Cloudinary** - бесплатно до 25 ГБ, автоматическая оптимизация

### Для продакшена:
1. **Код на GitHub Pages или Netlify**
2. **Медиафайлы на AWS S3 + CloudFront** - дешево и быстро

## Оптимизация медиафайлов

Перед загрузкой стоит оптимизировать:
- **Изображения**: сжать до разумных размеров (например, максимум 1920px по ширине)
- **Видео**: конвертировать в современные форматы (WebM, MP4 с H.264)
- **Lazy loading**: уже реализован в коде

## Настройка GitHub Pages

1. Создайте репозиторий на GitHub
2. Загрузите код (без медиафайлов):
   ```bash
   git remote add origin https://github.com/ваш-username/portfolio.git
   git push -u origin main
   ```
3. В настройках репозитория включите GitHub Pages
4. Выберите ветку `main` и папку `/ (root)`

## Настройка Cloudinary (пример)

1. Зарегистрируйтесь на cloudinary.com
2. Загрузите медиафайлы
3. Обновите `process_data.py` для генерации URL Cloudinary вместо локальных путей
