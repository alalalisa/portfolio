# Инструкция по исправлению путей Cloudinary

## Проблема
Cloudinary переименовал файлы, добавив суффиксы (например, `250_abc123` вместо `250`). Из-за этого файлы не загружаются по текущим URL.

## Решение

### Вариант 1: Автоматическая синхронизация (рекомендуется)
После сброса лимита API (10:00 UTC) запустите:

```bash
python sync_cloudinary_names.py
```

Этот скрипт:
1. Загрузит все файлы из Cloudinary
2. Создаст карту соответствий номеров и реальных имен
3. Обновит `portfolio_data.json` с правильными URL

### Вариант 2: Ручная проверка (если нужно срочно)
1. Откройте [Cloudinary Media Library](https://console.cloudinary.com/console/media_library)
2. Проверьте несколько файлов (например, 1, 50, 100, 250) в папках `images/` и `icons/`
3. Запишите паттерн именования (например, `250_abc123` или `250-xyz`)
4. Обновите скрипт `update_paths_with_suffixes.py` с этим паттерном

### Вариант 3: Перезагрузка файлов (если есть доступ к исходникам)
Если у вас есть исходные файлы, можно перезагрузить их с правильными `public_id`:

```bash
python reupload_with_correct_names.py
```

Этот скрипт перезагрузит все файлы, используя оригинальные имена без суффиксов.

После обновления `portfolio_data.json`:
1. Обновите страницу в браузере
2. Проверьте, что все изображения и видео загружаются
3. Проверьте консоль браузера на наличие ошибок загрузки
