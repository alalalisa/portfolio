# Быстрый старт

## Ответы на ваши вопросы

### 1. ✅ Версионирование настроено

Git репозиторий создан. Теперь вы можете:
- Сохранять каждую версию: `git add . && git commit -m "описание"`
- Откатываться: `git checkout <hash-коммита>`
- Экспериментировать в ветках: `git checkout -b experiment`

**См. `GIT_SETUP.md` для подробностей**

### 2. ✅ GitHub Pages - да, но с оговорками

**GitHub Pages отлично подходит для:**
- ✅ Хостинга кода сайта (HTML, CSS, JS)
- ✅ Бесплатно
- ✅ Автоматический деплой из Git
- ✅ Быстрая загрузка

**НО не подходит для:**
- ❌ 12 ГБ медиафайлов (слишком много)

**Решение:** Код на GitHub Pages, медиафайлы на внешнем сервисе.

### 3. ⚠️ 12 ГБ медиафайлов - нужно решить

**Проблема:** GitHub Pages не справится с таким объемом.

**Решения:**

#### Вариант A: Cloudinary (РЕКОМЕНДУЮ для начала)
- ✅ Бесплатно до 25 ГБ
- ✅ Автоматическая оптимизация
- ✅ Быстрый CDN
- ✅ Простая настройка

**Шаги:**
1. Зарегистрируйтесь на cloudinary.com
2. Загрузите медиафайлы
3. Обновите `prepare_for_hosting.py` с вашими настройками
4. Запустите скрипт для генерации URL

#### Вариант B: AWS S3 (для продакшена)
- ✅ Очень дешево (~$0.28/месяц для 12 ГБ)
- ✅ Надежно
- ✅ Быстро через CloudFront CDN
- ⚠️ Требует настройки

#### Вариант C: Netlify
- ✅ Можно загрузить медиа
- ✅ Бесплатный тариф: 100 ГБ трафика/месяц
- ⚠️ Но лучше все равно использовать внешнее хранилище

## Что делать сейчас

### Для экспериментов с дизайном:
1. Настройте Git (см. `GIT_SETUP.md`)
2. Делайте изменения
3. Сохраняйте версии: `git commit -m "описание"`
4. При неудаче: `git checkout <предыдущий-коммит>`

### Для подготовки к хостингу:
1. Выберите сервис для медиа (рекомендую Cloudinary)
2. Загрузите медиафайлы
3. Обновите `prepare_for_hosting.py` с URL вашего хранилища
4. Запустите скрипт
5. Загрузите код на GitHub Pages
