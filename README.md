# Портфолио медиахудожника

Современный веб-сайт для демонстрации портфолио медиахудожника с поддержкой фото и видео.

## Структура проекта

- `index.html` - главная страница портфолио
- `styles.css` - стили сайта
- `script.js` - JavaScript для динамической загрузки и отображения работ
- `process_data.py` - скрипт для обработки Excel файла и генерации JSON данных
- `portfolio_data.json` - данные портфолио (генерируется автоматически)
- `images/` - папка с медиафайлами (фото и видео)
- `сайт_портфолио.xlsx` - таблица с названиями и описаниями работ

## Установка и запуск

1. Убедитесь, что установлен Python 3.x
2. Установите необходимые библиотеки:
   ```bash
   pip install pandas openpyxl
   ```

3. Обработайте данные из Excel файла:
   ```bash
   python process_data.py
   ```

4. Откройте `index.html` в браузере или используйте локальный сервер:
   ```bash
   # Python
   python -m http.server 8000
   
   # Или Node.js
   npx http-server
   ```

5. Откройте браузер и перейдите по адресу `http://localhost:8000`

## Особенности

- ✅ Современный адаптивный дизайн
- ✅ Фильтрация по типу медиа (фото/видео)
- ✅ Модальное окно для просмотра работ в полном размере
- ✅ Автоматическое связывание медиафайлов с данными из Excel по номерам
- ✅ Поддержка различных форматов медиа (JPG, PNG, MP4, MOV, HEIC)
- ✅ Ленивая загрузка изображений для оптимизации производительности
- ✅ Автовоспроизведение видео при наведении в галерее

## Как это работает

1. Скрипт `process_data.py` читает Excel файл и извлекает номера из имен медиафайлов
2. Каждый медиафайл связывается с соответствующей строкой в Excel (номер в имени файла = номер строки)
3. Данные сохраняются в JSON формате
4. JavaScript загружает JSON и динамически создает галерею работ
5. При клике на работу открывается модальное окно с полной информацией

## Обновление данных

При добавлении новых работ:
1. Добавьте медиафайлы в папку `images/` с номерами в имени
2. Добавьте соответствующие строки в Excel файл
3. Запустите `python process_data.py` для обновления JSON
4. Обновите страницу в браузере

## Хостинг

**Важно:** Медиафайлы весят ~12 ГБ, что слишком много для GitHub Pages.

**Рекомендуемый подход:**
- **Код сайта** → GitHub Pages (бесплатно)
- **Медиафайлы** → Внешнее хранилище (Cloudinary, AWS S3, и т.д.)

**Варианты:**
1. **Cloudinary** - бесплатно до 25 ГБ, автоматическая оптимизация
2. **AWS S3 + CloudFront** - очень дешево (~$0.28/месяц для 12 ГБ)
3. **Netlify** - можно загрузить медиа, но лучше использовать внешнее хранилище

Подробнее см. `HOSTING_GUIDE.md`
